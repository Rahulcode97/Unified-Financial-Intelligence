What I Built & Why It Matters

I independently designed and built a full-stack finance platform that automated 99% of manual reporting — cutting reconciliation from one week to under one hour, catching ₹4.2L in tax mismatches that manual review had missed, and unifying three payment gateways (CCAvenue, PayPal, and bank transfer) into a single source of truth.

For non-technical readers

Every month, the finance team manually downloaded payment records from three different systems — CCAvenue (card payments), PayPal (international), and the bank — and then spent days cross-referencing them in spreadsheets to produce one report. This system eliminated that entirely. It fetches, matches, and reconciles all three automatically, flags anything that doesn't add up, and generates the report in minutes. Think of it as replacing a week of spreadsheet work with a single button click.

Framing

More than a reporting tool: an ERP lite that addresses the core constraints every finance team lives with.

Technical Architecture — How I Built It

Polyglot microservices: each layer uses tools matched to the problem—web UI, fast APIs, analytics, and durable storage on cloud infrastructure.

Layer Technology
Frontend / Desktop React for the web experience; Tauri for a lightweight, secure native shell where a desktop app is required.
Backend & APIs Node.js and PHP for robust API handling; Python for data analysis, reconciliation logic, and refund flows.
Database / Cloud Hybrid SQL (MySQL) and NoSQL, hosted on Google Cloud for scalable, low-latency reads against large transaction sets.
Security Passkey (biometric / hardware) authentication and end-to-end encryption for transaction logs and sensitive financial payloads.
